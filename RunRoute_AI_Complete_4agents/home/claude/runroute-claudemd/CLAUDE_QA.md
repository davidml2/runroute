# RunRoute AI — QA 에이전트

## 내 역할
데모 버전 품질 보증, 테스트 케이스 작성, 버그 리포트, 회귀 테스트, 릴리즈 체크리스트 관리

---

## 프로젝트 개요
AI 기반 러닝 루트 추천 앱 (iOS / Android / Apple Watch / WearOS)
현재 단계: **데모 버전 준비 중**

## 플랫폼 (테스트 대상)
- iOS (SwiftUI) — iPhone 12 mini ~ 15 Pro Max, iOS 16+
- Android (Jetpack Compose) — Galaxy S21~S24, Pixel 6~8, Android 12+
- Apple Watch (watchOS SwiftUI) — Series 6 ~ Ultra 2
- WearOS (Compose) — Galaxy Watch 4~6, Pixel Watch 1~2

---

## 🔴 데모 필수 테스트 (이게 깨지면 데모 망함)

### TC-001 메인 플로우 E2E
```
1. 앱 실행 → 현재 위치 자동 감지
2. 거리 5km / 지형 강변 선택
3. AI 루트 추천 3개 수신 (10초 이내)
4. 루트 카드 → 지도에 경로 표시
5. 내비게이션 시작 → 실시간 위치 추적
6. 러닝 완료 → 결과 화면 (거리/시간/페이스)
기대결과: 각 단계 오류 없이 전환
```

### TC-002 자연어 입력
```
입력: "강변 따라 5km 가볍게"
기대: 강변 지형 / 5km / 쉬움 난이도로 파싱
입력: "남산 한 바퀴"
기대: mountain 태그 / 5~7km 추천
```

### TC-003 Apple Watch 연동
```
- 폰 러닝 시작 → 워치 화면 자동 전환 확인
- 워치에서 심박수 실시간 표시 (0 or 999 아닌지)
- 워치 진행률 링 업데이트
- 러닝 완료 햅틱 피드백
```

### TC-004 결과 화면
```
- 거리 / 시간 / 페이스 / 심박수 정상 표시
- HealthKit 기록 저장 (iOS)
- Google Fit 기록 저장 (Android)
```

## 🟡 데모 중 질문 대비 시나리오

### TC-005 GPS 없을 때
```
조건: 위치 권한 거부 or 실내
기대: 친절한 안내 메시지 + 수동 위치 입력 옵션
NG:  앱 크래시 or 빈 화면
```

### TC-006 워치 미연결
```
조건: Apple Watch 없이 iPhone만
기대: 폰 단독으로 모든 기능 작동
NG:  워치 관련 오류 팝업 노출
```

### TC-007 AI 응답 지연
```
조건: 네트워크 느림 (3G 수준)
기대: 로딩 애니메이션 + "AI 분석 중..." 표시
시간: 최대 15초 내 응답 or 타임아웃 메시지
NG:  무한 로딩 or 화면 멈춤
```

### TC-008 러닝 중 전화 수신
```
조건: 내비게이션 중 전화 수신
기대: 통화 후 앱 복귀 시 내비 정상 재개
NG:  기록 초기화 or 앱 종료
```

### TC-009 백그라운드 위치 추적
```
조건: 화면 잠금 상태로 10분 러닝
기대: 위치 추적 유지, 거리 정상 누적
기기: iOS (백그라운드 모드), Android (포그라운드 서비스)
```

## 🟢 엣지케이스 (데모 이후)

### TC-010 AI 이상 입력
```
- "화성에서 5km" → 현재 위치 기반으로 폴백
- 빈 입력 "" → 기본값(5km/공원) 적용
- 특수문자 "!@#$%" → 에러 없이 처리
- 매우 긴 문장 (200자+) → 핵심만 파싱
```

### TC-011 루트 이탈 감지
```
기준: 경로에서 50m 이상 벗어날 때
기대: 워치 햅틱 + "이탈" 경고 + 자동 재탐색
오발동 테스트: GPS 오차(30m)에서 이탈 안 뜨는지
```

### TC-012 장시간 러닝
```
조건: 30분 이상 연속 러닝
체크: 메모리 누수 없는지 (iOS Instruments)
체크: 배터리 소모율 (시간당 15% 이내 목표)
체크: 위치 정확도 유지
```

### TC-013 심박수 이상값
```
센서 오류: 0 bpm → "--" 표시
센서 오류: 999 bpm → 필터링 후 이전값 유지
연결 끊김: 워치 심박 없을 때 폰 화면 처리
```

## 버그 리포트 양식

버그 발견 시 개발 에이전트에게 이 형식으로 전달:

```
[BUG-###] 제목
플랫폼: iOS 17 / iPhone 15 Pro
심각도: Critical / High / Medium / Low
재현율: 항상 / 가끔 (3/5회) / 드물게

재현 단계:
1.
2.
3.

기대 결과:
실제 결과:
스크린샷/영상: (첨부)
```

### 심각도 기준
- **Critical**: 앱 크래시, 데이터 유실, 메인 플로우 불가
- **High**: 주요 기능 오작동, 데모에서 노출 가능
- **Medium**: 부분 기능 오류, 회피 방법 있음
- **Low**: UI 깨짐, 오탈자, 미세한 UX 불편

## 데모 전 최종 체크리스트

```
□ TC-001 ~ TC-004 모두 PASS
□ Critical / High 버그 0개
□ 시뮬레이터 + 실기기 모두 확인
□ 앱스토어/플레이스토어 심사 규정 위반 없음
□ 개인정보 처리방침 화면 있음
□ HealthKit / 위치 권한 설명문 자연스러운지
□ 네트워크 없을 때 오류 메시지 있음
□ 데모 시나리오 3회 연속 오류 없이 통과
```
